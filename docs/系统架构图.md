## 三、请求数据流向（典型场景）

### 场景 1：用户登录

```mermaid
sequenceDiagram
    actor U as 👤 用户
    participant MP as 📱 微信小程序
    participant WX as 🔐 微信开放接口
    participant API as ☁️ Express API
    participant DB as 🗄️ SQLite

    U->>MP: 点击「微信一键登录」
    MP->>WX: wx.login() 获取临时 code
    WX-->>MP: code
    MP->>API: POST /auth/login {code}
    API->>WX: code2Session 换取 openid
    WX-->>API: openid + session_key
    API->>DB: INSERT OR IGNORE INTO users (openid)
    API->>DB: SELECT * FROM users WHERE openid=?
    DB-->>API: user 记录
    API->>API: 签发 JWT Token (payload: user_id, openid)
    API-->>MP: {code:0, data:{token, nickname, avatar_url}}
    MP->>MP: wx.setStorageSync('token', token)
    MP-->>U: 进入首页
```

### 场景 2：睡眠报告查看（含模拟数据生成）

```mermaid
sequenceDiagram
    actor U as 👤 用户
    participant MP as 📱 微信小程序
    participant API as ☁️ Express API
    participant SIM as ⚙️ 模拟数据生成器
    participant DB as 🗄️ SQLite

    Note over SIM,DB: ⏰ 每日凌晨 2:00 自动运行
    SIM->>DB: 读取 user_settings (bedtime, wakeup_time)
    SIM->>SIM: 按生理规律生成分期/心率/呼吸/噪音数据
    SIM->>DB: INSERT INTO sleep_reports (...)

    U->>MP: 打开「睡眠报告」页
    MP->>API: GET /report/daily?date=2026-06-29<br/>Authorization: Bearer <token>
    API->>API: JWT 中间件校验 → 提取 user_id
    API->>DB: SELECT * FROM sleep_reports<br/>WHERE user_id=? AND report_date=?
    alt 数据存在
        DB-->>API: 返回报告记录
        API->>API: 解析 stage_curve / noise_curve JSON
        API-->>MP: {stages:[], noiseCurve:[], score, ...}
        MP->>MP: ECharts 渲染分期柱状图 + 噪音折线图
        MP-->>U: 展示完整报告
    else 数据不存在
        API->>SIM: 实时生成指定日期数据
        SIM->>DB: INSERT
        DB-->>API: 返回新报告
        API-->>MP: 同上
    end
```

### 场景 3：周趋势查询

```mermaid
sequenceDiagram
    actor U as 👤 用户
    participant MP as 📱 微信小程序
    participant API as ☁️ Express API
    participant DB as 🗄️ SQLite

    U->>MP: 切换到「周」视图
    MP->>API: GET /report/history?type=week&end=2026-06-29
    API->>DB: SELECT report_date, sleep_score, total_minutes,<br/>deep_minutes, rem_minutes, light_minutes<br/>FROM sleep_reports<br/>WHERE user_id=? AND report_date BETWEEN ? AND ?
    DB-->>API: 近 7 天记录
    API->>API: 聚合计算：日均分、深睡比例、趋势值
    API-->>MP: {daily:[{date,score,duration,deepPct}], ...}
    MP->>MP: ECharts 渲染折线图
    MP-->>U: 展示评分趋势 + 时长柱状图
```

### 场景 4：医生授权 → 查看脱敏报告

```mermaid
sequenceDiagram
    actor P as 👤 患者
    actor D as 🩺 医生
    participant MP as 📱 小程序
    participant Web as 🌐 医生端 Web
    participant API as ☁️ Express API
    participant DB as 🗄️ SQLite

    Note over P,DB: ─── 步骤 A：患者发起授权 ───
    P->>MP: 输入医生手机号，发起授权
    MP->>API: POST /doctor/authorize {doctorPhone, expireDays:30}
    API->>DB: INSERT INTO doctor_authorizations<br/>(patient_user_id, doctor_user_id, expire_at, status='active')
    API-->>MP: {authCode, expireAt}
    MP-->>P: 提示「已发起授权」

    Note over D,DB: ─── 步骤 B：医生查看报告 ───
    D->>Web: 登录医生端
    Web->>API: POST /doctor/login {phone, password}
    API-->>Web: JWT Token

    D->>Web: 点击患者 A（匿名 ID）
    Web->>API: GET /doctor/patient/report?patientId=X&date=...
    API->>DB: 校验 doctor_authorizations 有效性<br/>(status=active AND expire_at > now)
    API->>DB: SELECT 脱敏字段 FROM sleep_reports<br/>WHERE user_id=X (不含真实姓名、手机号)
    DB-->>API: 脱敏报告数据
    API-->>Web: {anonymousId, sleepScore, stages, ...}
    Web-->>D: 展示脱敏睡眠报告
```

## 五、实训阶段部署拓扑

```mermaid
graph TB
    subgraph 开发机["🖥️ 实训开发机 (Windows 11)"]
        direction TB
        EX["Express Server<br/>localhost:3000"]
        SQL["SQLite<br/>./data/sleep_care.db"]
        WEB2["医生端 Web<br/>localhost:3000/doctor<br/><i>Express 托管静态资源</i>"]
        EX --- SQL
        EX --- WEB2
    end

    WXDEV["📱 微信开发者工具<br/>—— 不校验合法域名 ——"]
    WXDEV -.->|"wx.request<br/>http://localhost:3000"| EX

    style 开发机 fill:#e8f5e9,stroke:#4caf50,stroke-width:2px
```

> **说明**：实训阶段所有服务集中部署在同一台开发机上。微信小程序通过开发者工具的「不校验合法域名」选项直连本地 `localhost:3000` API；医生端 Web 页面作为 Express 静态资源托管，无需额外启动 Web 服务器。后续正式上线时，仅需将 Express 部署到云服务器、SQLite 替换为 MySQL/PostgreSQL、配置 HTTPS 域名白名单即可。
