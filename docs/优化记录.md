## 1. 首页加载速度优化

### 优化前的问题

| 现象 | 根因 |
|------|------|
| 每次进入首页显示 "加载中..."，等待约 500ms | `onLoad` 和 `onShow` 每次无条件设置 `loading: true`，丢弃已有数据 |
| 从子页面返回首页时闪白屏 | WXML `wx:if="{{loading}}"` 直接显示空白加载页，即使有缓存数据也不展示 |
| 5 分钟内多次进出首页，每次都要等待 | 无数据缓存机制，每次必调 API |

```
用户：首页 → 报告页 → 返回首页
              ↓
     loading:true → "加载中..." → API 返回 → 渲染
     （每次 ~500ms 白屏等待）
```

### 优化方案

采用 **localStorage 缓存 + 静默刷新** 策略：

1. 首次加载时 (`onLoad`)：先读取 5 分钟缓存直接渲染 → 再调 API 后台更新
2. 再次进入时 (`onShow`)：已有数据不设 `loading`，后台静默请求最新数据
3. 缓存过期时：正常显示加载态，重新请求

| 缓存键 | 内容 | 有效期 |
|--------|------|--------|
| `home_report_cache` | 睡眠报告数据 | 5 分钟 |
| `home_notes_cache` | 医生建议列表 | 5 分钟 |

### 涉及文件

- `miniprogram/pages/home/home.js` — 新增 3 个缓存工具方法 + 修改 `onLoad`/`onShow`/`loadReport`/`loadNotes`

### 优化后的效果

| 场景 | 优化前 | 优化后 |
|------|--------|--------|
| 首次打开首页 | ~500ms "加载中..." | ~500ms "加载中..."（无缓存，不变） |
| 5 分钟内再进入 | ~500ms "加载中..." | **即时渲染** 缓存数据，不闪白屏 |
| 从报告页返回首页 | ~500ms 白屏 | **即时渲染**，后台更新数据 |
| 缓存过期后进入 | — | 正常 "加载中..." + API 刷新 |
| pull-to-refresh | 刷新 | 强制刷新 + 更新缓存 |
