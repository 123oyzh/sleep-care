# 医患通知方案设计

> 基于实训项目"智能睡眠环境调控设备"V1.0，为医患双方提供医生干预建议的通知与查阅功能。
> 采用"轮询 + 红点提示"方案，无需外部推送服务，适合实训阶段快速实现。

---

## 一、背景与需求

### 当前状态

| 功能 | 状态 |
|------|------|
| 医生 Web 端填写干预建议（`PUT /api/doctor/note`） | ✅ 已实现 |
| 建议存入 `doctor_authorizations.doctor_note` | ✅ 已实现 |
| 患者端读取建议 | 缺失 |
| 患者端接收新建议通知 | 缺失 |

### 需求目标

1. 医生填写干预建议后，患者能收到"有新建议"的通知
2. 患者在小程序中能查看医生给自己写的完整建议
3. 方案简单可行，适合实训项目，不引入外部服务

---

## 二、方案选型

### 方案对比

| 方案 | 原理 | 优点 | 缺点 | 推荐 |
|------|------|------|------|------|
| **A. onShow 刷新 + 通知条** | 依赖页面切换时 `onShow` 生命周期刷新数据 | 零额外开销，实现最简单 | 依赖用户主动切换页面 | ❌ 患者可能不切换页面 |
| **B. 轮询 + 红点提示** | `setInterval` 每 30 秒请求轻量状态接口 | 实时性好，红点直观，不依赖用户操作 | 需管理定时器生命周期 | ✅ 推荐 |
| **C. WebSocket 推送** | 长连接实时推送 | 真正实时 | 需额外 WebSocket 服务，复杂度高 | 实训阶段不适用 |

### 最终方案：轮询 + 红点提示

```
医生 Web 端写入建议 → note_updated_at 时间戳
                        ↓
患者小程序首页 ← 每 30 秒 → GET /api/patient/notes/status
                        ↓
              比较 localStorage.last_read_at
                        ↓
              新建议? → 🔴 红点数字
                        ↓
              点击 → 通知页 → 标记已读
```

## 三、核心设计

### 3.1 未读判断逻辑

```
unread = (latest_note_updated_at > localStorage.note_last_read_at) ? total : 0
```

- `latest_note_updated_at`：服务端返回的最新建议更新时间
- `note_last_read_at`：患者最后一次进入通知页的时间，存储在 `wx.setStorageSync`
- 进入通知页时自动写入当前时间 → 红点消失

### 3.2 轮询生命周期

```
onLoad / onShow → startPolling() → loadNotesStatus() + setInterval(30s)
onHide / onUnload → clearInterval → 停止轮询
```

- 仅在首页可见时轮询，离开页面自动停止
- 避免多个页面同时轮询造成重复请求

## 四、API 接口定义

### 4.1 现有接口（复用）

#### GET /api/patient/notes —— 获取完整建议列表

```
请求头: Authorization: Bearer <token>
响应:
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "doctor_id": 3,
      "doctor_name": "张医生",
      "doctor_phone": "138****8001",
      "note": "建议保持规律作息，睡前避免饮用咖啡和浓茶",
      "note_updated_at": "2026-07-02 14:30:00"
    }
  ]
}
```

**后端实现**：`backend/app.js`，JOIN `doctor_authorizations` + `users`，条件 `status='active' AND doctor_note IS NOT NULL`

#### PUT /api/doctor/note —— 医生写入建议（已修改）

```
请求体: { "patient_id": 1, "note": "建议..." }
响应:   { "code": 0, "message": "建议已保存", "data": { "patient_id": 1, "note": "建议..." } }
```

**修改点**：更新 SQL 增加 `note_updated_at = ?`，记录每次写入时间。

### 4.2 新增接口

#### GET /api/patient/notes/status —— 轻量状态检查（供轮询使用）

```
请求头: Authorization: Bearer <token>
响应:
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 1,
    "latest_note_updated_at": "2026-07-02 14:30:00",
    "latest_doctor_name": "张医生"
  }
}
```

**设计意图**：只返回计数和时间，不返回建议正文，控制带宽。轮询调用此接口，用户点击时再调用 `/api/patient/notes` 获取完整内容。

**后端实现**：`backend/app.js`，子查询获取最新医生名。

## 五、数据库变更

### 5.1 新增列

```sql
ALTER TABLE doctor_authorizations ADD COLUMN note_updated_at TEXT;
```

| 列名 | 类型 | 说明 |
|------|------|------|
| `note_updated_at` | TEXT | 医生最后一次修改建议的时间。与 `updated_at` 分离，避免授权状态变更产生误报 |

**迁移策略**：在 `backend/db/schema.js` 中使用 `try { db.run("ALTER TABLE ...") } catch(_) {}` 模式，与 `noise_json`、`sleep_stages_json` 等列一致。

## 六、小程序实现

### 6.1 文件结构

```
miniprogram/pages/
├── home/          # 首页 — 轮询 + 红点展示
│   ├── home.js    # 新增: startPolling(), loadNotesStatus(), goToNotify()
│   ├── home.wxml  # 新增: note-red-dot 红点组件
│   └── home.wxss  # 新增: .note-red-dot 样式
└── notify/        # 通知页 — 新建
    ├── notify.js      # loadNotes() + _markRead()
    ├── notify.wxml    # 医生建议卡片列表
    ├── notify.wxss    # 卡片样式
    └── notify.json    # {"navigationBarTitleText":"医生建议"}
```

### 6.2 首页改造要点

**home.js**：
- `data.unreadCount: 0` — 红点数字
- `startPolling()` — 启动 30 秒轮询，先立即执行一次
- `loadNotesStatus()` — 调用 `/api/patient/notes/status`，计算未读数
- `onHide()` / `onUnload()` — 清除定时器
- `goToNotify()` — 跳转通知页

**home.wxml**：
```xml
<view class="note-banner" wx:if="{{hasNotes}}" bindtap="goToNotify">
  <text class="note-banner-icon">🔔</text>
  <text class="note-banner-text">您有 {{notes.length}} 条医生建议，点击查看</text>
  <view class="note-red-dot" wx:if="{{unreadCount > 0}}">
    <text>{{unreadCount}}</text>
  </view>
  <text class="note-banner-arrow">→</text>
</view>
```

### 6.3 通知页要点

**notify.js**：
- `onLoad()` 中调用 `loadNotes()`（完整建议）+ `_markRead()`
- `_markRead()` 写入 `wx.setStorageSync('note_last_read_at', now)` → 红点消失
- 点击卡片跳转到报告页查看完整上下文

**notify.wxml**：每条建议显示：
- 医生头像 💬 + 姓名（蓝色）+ 脱敏手机号
- 建议内容正文（灰色，多行）
- 更新时间（小字灰色）
- "点击查看完整报告 →" 引导

### 验证步骤

1. 患者登录小程序 → 首页无红点
2. 医生 Web 端确认授权 + 填写干预建议 + 保存
3. 等待 30 秒（或手动 pull-to-refresh）→ 首页出现 🔴 红点
4. 点击通知条 → 进入通知页 → 红点消失 → 看到建议完整内容
5. 点击卡片 → 跳转报告页 → 底部看到医生建议
6. 再次返回首页 → 红点不再显示（已标记已读）
